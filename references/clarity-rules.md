# Clarity Rules

The enemy is AI clutter: many symbolic objects, no readable relation.

## Clarity Budget

One image should usually have:

- 1 reader takeaway
- 1 visual structure
- 1 metaphor world
- 1 focal action
- 1 main connector grammar
- labels chosen by diagram form and cognitive load
- 5-9 objects
- 1-2 accent colors

If you need more, split the image.

## Three-Anchor Rule

Every image needs three anchors:

1. Start: where the reader begins.
2. Change: the mechanism, conflict, boundary, or transformation.
3. Result: where the idea lands.

If these anchors are not visible, the image is not clear.

## Object Test

Each object must answer one question:

- What concept does it represent?
- What relationship does it show?
- What would break if it were removed?

If no answer, remove it.

## Metaphor Test

A metaphor is good when it makes structure clearer.

Bad metaphor:

- interesting but unrelated
- needs explanation
- adds extra symbolism
- makes the article feel more mysterious

Good metaphor:

- maps abstract relation to physical relation
- shows cause, boundary, sequence, weight, or comparison
- can be explained in one sentence

## Reader Path

The viewer's eye should move in one of these patterns:

- left -> right
- top -> bottom
- outside -> inside
- center -> parts
- repeated panels left -> right
- loop clockwise

Do not make the reader hunt around randomly.

## Anti-Clutter Prompts

Use when a prompt is getting too symbolic:

```text
Use fewer objects. Make one clear start point, one visible transformation, and one result. Remove decorative metaphors that do not affect the idea.
```

Use when the image has many elements but no relation:

```text
Make the relationship between objects physically explicit using one connector grammar: path, boundary, containment, weight, repetition, transformation, or causality.
```

Use when the image looks clever but unreadable:

```text
Reduce the metaphor to one physical action. The viewer should understand start, change, and result in three seconds.
```
