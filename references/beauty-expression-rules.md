# Beauty And Expression Rules

## Hierarchy

Every image needs:

1. one focal object or action
2. one supporting path/relationship
3. a few labels
4. a quiet area

If everything has equal weight, the image will feel like a slide or AI clutter.

## Composition

- Start with a 6-column or 8-column editorial grid.
- Put the focal structure across 3-5 columns, not full width by default.
- Keep one side or the top third quieter.
- Use modular alignment to make the image feel designed.
- Use overlap, containment, scale, or sequence to imply relation, not only arrows.
- Use one large object and several small objects, not many medium objects.
- Avoid symmetrical balance unless the structure is comparison or tradeoff.
- Choose a diagram form that naturally uses the canvas: pyramid for vertical support, matrix for fields, map for territory, loop for return, cutaway for hidden parts, storyboard for human change.
- Use gutters, margins, and pause zones as part of the reading path.
- Prefer rounded, hand-friendly, editorial modules over hard square boxes unless the form needs technical rigidity.
- Replace plain triangles with soft layered stacks, cut-paper pyramids, sediment shelves, or stepped terrain.
- Use spatial depth only when it clarifies hierarchy, distance, containment, route, or hidden mechanism.

## Expressiveness

The metaphor should be a little surprising but physically understandable. Prefer object systems over mascot scenes:

- an idea being squeezed through a filter
- a sentence fermenting in a jar
- a decision sitting on a scale
- a workflow jammed in one slot
- a reader crossing stepping stones
- a trend becoming weather

The visual should not become surreal nonsense. The action must map to the article's idea.

## Distinctness From Xiaohei-Like Skills

Default away from:

- whiteboard doodle composition
- cute small character doing the action
- loose handwritten labels everywhere
- black line art as the whole identity
- whimsical mascot absurdity

Default toward:

- editorial grid
- typographic structure
- paper-cut/vector-collage modules
- object-first explanation
- chart and diagram hybrids
- designed label zones
- shared visual tokens: semantic colors, line weights, line patterns, label containers, and spatial rules
- soft diagram forms with clear whitespace and one small visual twist

## Label Placement

- Labels should be part of the world: tag, stamp, sign, sticky note, tray label, caption.
- Keep labels near the object they name.
- Use the article's own vocabulary.
- Use nouns and short verbs, not explanations.

## Style Consistency Across An Article

For a package of images:

- keep the same background
- keep the same line weight
- keep the same label style
- reuse one metaphor world or compatible worlds
- vary structures so images do not feel repetitive

Example: a product essay might use workshop + manual diagram + paper object theater. A personal essay might use miniature world + comic beat + field notes.

Across an article package, keep form variety intentional:

- one flow/map image for movement
- one pyramid/stack/cutaway image for structure
- one matrix/scorecard/small-multiple image for choice or comparison
- one storyboard/room/picture-book image for the human consequence

Do not make every image the same row of steps.

## Regeneration Priority

Regenerate before minor editing when:

- structure is wrong
- metaphor world is wrong
- focal action is missing
- image looks like PPT
- characters are decorative
- too many labels are required to understand it

Edit when:

- text is wrong
- one object is off
- spacing/crop is slightly wrong
- a title needs removal
