# Diagram Form Catalog

Use this catalog after article analysis and before style selection. It is the bridge between "what the article means" and "what the image should look like".

The form is not decoration. The form is the reader's thinking path. Because image generation can render complex spatial scenes, do not limit the skill to flat PPT diagrams, Mermaid-like boxes, or simple arrows.

## Selection Questions

Pick the form by answering these in order:

1. What relation must the reader understand first?
2. Is the article explaining sequence, hierarchy, comparison, territory, proportion, causality, accumulation, spatial relation, or a human situation?
3. Does the image need data, conceptual structure, physical metaphor, or spatial scene?
4. What is the simplest form that preserves the article's point?
5. What form would make the page feel beautiful instead of like a training slide?

If two forms compete, choose the one that makes the reader's first three seconds clearer. Use hybrid forms only when the hierarchy is explicit.

## Core Form Families

### 1. Flow / Process

Use for: workflows, operating procedures, transformation chains, agent handoffs, content pipelines, life routines.

Forms: linear flow, curved path, conveyor, handoff chain, branching decision flow, swimlane flow, before-during-after sequence.

Best layout: one visible path with 3-5 stations. Use rounded stations, object stages, or physical work surfaces instead of identical square boxes.

Beauty move: replace arrow soup with a ribbon, track, road, cord, shelf, pipe, or light trail.

Avoid: more than 6 steps, equal boxes with equal arrows, checkmarks under every step, generic SaaS process slide composition.

Prompt token: `one graceful visible process path with 3-5 physical stations, each station changes the object`.

### 2. Pyramid / Layered Hierarchy

Use for: priority, maturity, needs, capability levels, foundations, strategic layers, stacked causes.

Forms: pyramid, stepped pyramid, layered stack, foundation tower, tier shelf, mountain contour, sediment cross-section.

Best layout: 3-5 levels. The base must visibly support the top. Top label should be small but important; base label should feel structural.

Beauty move: soften the geometry with rounded strata, cut-paper layers, sediment, shelves, glass slabs, or stepped terrain.

Avoid: corporate triangle clipart, too many levels, centered title plus bullet labels, decorative 3D bevels.

Prompt token: `soft editorial pyramid or layered foundation stack with 3-5 labeled strata, base visibly supports the top`.

### 3. Matrix / Quadrant

Use for: tradeoffs, positioning, decision spaces, strategy options, product categories, taste maps.

Forms: 2x2 matrix, spectrum map, opportunity grid, risk-value grid, effort-impact grid, constraint map.

Best layout: two axes with strong semantic names. Place 4-8 items as tokens, not long cards.

Beauty move: use soft field colors or paper zones; make one quadrant the visual focus if the article has a recommendation.

Avoid: unlabeled axes, dumping every idea into the grid, many colored dots with no story.

Prompt token: `clean editorial 2x2 decision matrix with semantic axes, 4-8 positioned tokens, one highlighted quadrant`.

### 4. Map / Territory

Use for: journeys, migration, learning paths, market terrain, conceptual territory, project roadmaps, life transitions.

Forms: route map, island map, subway map, terrain contour, neighborhood map, migration path, heat territory.

Best layout: one path or one territory with 3-6 landmarks. The map should show choice, obstacle, distance, or destination.

Beauty move: keep empty territory as breathing space; make landmarks concrete and distinct.

Avoid: dense roadmap boards, random decorative landmarks, map pins everywhere, visible legend larger than the actual story.

Prompt token: `clean editorial route map with 3-6 meaningful landmarks, visible destination, generous empty territory`.

### 5. Timeline / Calendar

Use for: history, launch plan, habit loop over time, aging, learning curve, before and after moments.

Forms: horizontal timeline, stacked timeline, calendar strip, arc timeline, milestone shelf, time spiral.

Best layout: 3-6 moments. Show what changes across time, not just dates.

Beauty move: use quiet date labels and one strong turning point; vary the object state across moments.

Avoid: too many date labels, generic milestone circles, Gantt clutter unless the article is about project scheduling.

Prompt token: `timeline as a calm editorial shelf, 3-6 moments, one visible turning point and changed object state`.

### 6. Loop / Flywheel

Use for: feedback systems, habits, compounding, product growth, learning, recursive agent loops.

Forms: feedback loop, flywheel, spiral, loop with residue, circular service cycle, compounding ring.

Best layout: one entry point, 3-5 loop stages, one accumulated output. Make the return path clear.

Beauty move: make the loop tactile, not a sterile circle; show residue, memory, or compounding as visible accumulation.

Avoid: circular arrows with no changed state, self-help flywheel cliche, too many stages.

Prompt token: `tactile feedback loop with entry point, 3-5 stages, visible return path, accumulated residue`.

### 7. Funnel / Filter / Bottleneck

Use for: selection, conversion, loss, compression, quality gates, constraint, hidden failure.

Forms: funnel, sieve, filter stack, narrow bridge, pressure valve, queue bottleneck, leakage pipe.

Best layout: input mass, local constriction, output result. Loss or residue must be visible.

Beauty move: use a physical device: sieve, gate, membrane, press, valve, sorting tray.

Avoid: generic conversion funnel, clean funnel with no meaning, symmetrical marketing slide shape.

Prompt token: `physical filter or bottleneck device, input mass enters, local constraint changes it, output and residue are visible`.

### 8. Anatomy / Cutaway

Use for: hidden mechanisms, systems, product internals, mental models, organizational machinery.

Forms: exploded view, cutaway block, cross-section, labeled mechanism, x-ray layer, parts tray.

Best layout: one main object opened just enough to reveal 4-7 parts.

Beauty move: make one hidden part the focal discovery; keep parts aligned to an invisible grid.

Avoid: parts floating everywhere, technical labels too small to read, decorative complexity.

Prompt token: `editorial cutaway diagram, one object opened to reveal 4-7 aligned internal parts with short attached callouts`.

### 9. Ecosystem / Network

Use for: stakeholders, dependencies, communities, platforms, agents, knowledge graphs, supply chains.

Forms: hub and spokes, constellation, dependency web, ecosystem garden, market map, orbit system.

Best layout: one focal node or one path through the network. Use clusters, not a hairball.

Beauty move: show clusters as neighborhoods, islands, shelves, or orbits; label clusters instead of every node.

Avoid: equal nodes connected everywhere, random graph with no read path, decorative constellation dots.

Prompt token: `organized ecosystem map with one focal node, 3-5 clusters, controlled connection weights, no graph hairball`.

### 10. Comparison / Small Multiples

Use for: options, failure modes, personas, versions, patterns, scenarios.

Forms: two-panel contrast, 3-card comparison, 2x2 small multiples, same-frame scenario row, wrong-right pair.

Best layout: repeat the same frame and change one variable per panel.

Beauty move: align panels precisely; use one highlighted difference.

Avoid: different camera angles per panel, long explanations, too many variables changing at once.

Prompt token: `small multiples, same frame repeated, one variable changes per panel, concise attached captions`.

### 11. Evidence / Table / Scorecard

Use for: argument with proof, criteria, audit, product teardown, recommendation, pros and cons.

Forms: mini table, scorecard, checklist card, evidence shelf, ranked slips, annotated receipts.

Best layout: 2-5 criteria and 2-4 subjects. The result should be readable without a paragraph.

Beauty move: turn cells into cards, stamps, slips, beads, or trays; highlight the decisive row or criterion.

Avoid: full spreadsheet, checkmark wallpaper, fake precision.

Prompt token: `editorial mini scorecard with 2-5 criteria, evidence slips or stamps, one decisive highlighted row`.

### 12. Proportion / Part-To-Whole

Use for: resource allocation, time use, attention split, composition, budget, workload.

Forms: stacked bar, waffle grid, nested containers, proportional shelf, packed circles, divided field.

Best layout: one whole clearly divided into parts. Label only the meaningful parts.

Beauty move: use tactile containers instead of default pie charts; keep parts grouped by meaning.

Avoid: pie chart by habit, too many slices, color-only meaning.

Prompt token: `one whole divided into meaningful parts using nested containers or proportional shelves, direct labels, no busy pie chart`.

### 13. Spectrum / Ladder / Scale

Use for: maturity, intensity, risk level, values, emotion, capability, gradual change.

Forms: ladder, gradient scale, stepping stones, thermometer, pressure gauge, maturity rail.

Best layout: one axis from low to high or old to new. Show where the article's subject sits.

Beauty move: use physical steps, rails, shelves, or pressure marks; mark current state and desired state.

Avoid: vague gradients without labels, 10-step maturity models, decorative gauges.

Prompt token: `editorial ladder or spectrum scale with current state, threshold marks, and desired state clearly placed`.

### 14. Storyboard / Comic Diagram

Use for: user situations, emotional states, personal essays, culture commentary, before-insight-after moments.

Forms: 2-panel beat, 3-beat strip, 2x2 explanation comic, picture-book spread, scene plus diagram overlay.

Best layout: 2-4 panels, one emotional or situational variable changes. Characters are optional and secondary.

Beauty move: use clear-line objects and simple props; put the insight in the last panel or overlay.

Avoid: manga/anime surface style, cute mascot as the main identity, speech bubble clutter.

Prompt token: `clear-line editorial storyboard, 2-4 panels, one situation changes, short captions, optional secondary abstract figures`.

### 15. Tree / Nested Hierarchy

Use for: taxonomies, nested concepts, org structures, category systems, file trees, product architecture.

Forms: tree diagram, icicle, nested boxes, treemap-like block, org chart, decision tree, branching root system.

Best layout: root plus 2-4 branches, or one nested container with clear parent-child relation.

Beauty move: turn branches into shelves, roots, folders, or nested paper compartments; direct labels on branch edges.

Avoid: dense mind-map hairball, too many hierarchy levels, tiny branch labels.

Prompt token: `clean nested hierarchy with root and 2-4 branches, parent-child relation visible, direct branch labels`.

### 16. Set / Overlap / Venn

Use for: overlap, shared responsibilities, competing definitions, category intersections, "both/and" arguments.

Forms: Venn, Euler diagram, overlapping fields, blended territories, shared room zone.

Best layout: 2-3 sets only. Label the overlap as the focal meaning.

Beauty move: use translucent paper fields or soft terrain overlaps; make the intersection tactile.

Avoid: 4+ circles, vague overlaps, decorative transparency with no claim.

Prompt token: `2-3 overlapping semantic fields with one highlighted intersection and direct labels`.

### 17. Flow Quantity / Sankey / Alluvial

Use for: quantity moving between categories, budget, energy, attention, audience migration, pipeline loss.

Forms: Sankey, alluvial, weighted route, braided stream, bundle flow.

Best layout: 3-5 source/target groups. Width means quantity only if true data exists.

Beauty move: turn flows into ribbons, braided cords, rivers, or material streams with direct labels.

Avoid: fake thickness, spaghetti ribbons, decorative flow width.

Prompt token: `weighted flow ribbon diagram, 3-5 grouped sources and targets, width only encodes true quantity`.

### 18. Service Blueprint / Journey Layers

Use for: user experience, operational systems, customer journey, frontstage/backstage processes, work/life routines.

Forms: service blueprint, journey map, experience lane, frontstage/backstage swimlane, emotional curve.

Best layout: timeline or route plus 2-4 horizontal layers: user action, system action, evidence, emotion.

Beauty move: use a layered path, transparent stage, or cutaway workspace with lanes that feel physical.

Avoid: huge consulting template, too many lanes, sticky-note overload.

Prompt token: `editorial service blueprint with a route and 2-4 layered lanes, frontstage/backstage relation visible`.

### 19. Causal / Systems Model

Use for: why something happens, reinforcing loops, balancing loops, root causes, system dynamics.

Forms: causal loop, stock-and-flow, root cause tree, force-field, leverage map.

Best layout: 3-6 variables with one main causal path. Mark reinforcing or balancing loops only if needed.

Beauty move: make causality physical: levers, reservoirs, pressure valves, roots, or feedback cords.

Avoid: arrows everywhere, abstract variables with no objects, implying certainty without evidence.

Prompt token: `causal system model with 3-6 variables, one main causal path, physical levers or reservoirs`.

### 20. Value Chain / Strategic Position

Use for: market structure, product strategy, moat, maturity vs visibility, platform ecosystems.

Forms: value chain, Wardley-like map, capability landscape, dependency ladder, strategic terrain.

Best layout: one axis for value/visibility/maturity and one path or chain. Show movement or pressure.

Beauty move: translate the map into terrain, shelves, or a workshop wall rather than a sterile strategy slide.

Avoid: unlabeled strategic axes, consultant-cliche boxes, fake precision.

Prompt token: `strategic landscape map with value chain path, semantic axes, movement pressure, calm editorial labels`.

### 21. Canvas / Framework Board

Use for: methods, operating systems, personal workflows, business models, strategy frameworks.

Forms: framework canvas, modular board, workshop wall, kanban-like map, operating system board, nested cards.

Best layout: 4-8 modules grouped into 2-3 zones. Show relationships between modules, not just a list.

Beauty move: use varied module sizes, labels as tabs or board headers, and one quiet margin.

Avoid: equal card wall, dense Notion dashboard look, decorative sticky notes everywhere.

Prompt token: `designed framework canvas with 4-8 varied modules grouped into zones, visible relationships, not an equal card wall`.

## Spatial / 3D Form Families

Use spatial forms when flat diagrams hide the point. These forms are allowed because image generation can render hierarchy, depth, occlusion, rooms, terrain, and object relationships.

### 22. Spatial Stack / 3D Hierarchy

Use for: deep hierarchy, nested responsibilities, capability architecture, layered life/work systems.

Forms: isometric tier stack, transparent layer tower, floating shelves, stepped terrain, room floors, nested platforms.

Best layout: 3-5 layers in perspective. Show vertical support, access paths, and what sits on each layer.

Beauty move: use isometric or gentle three-quarter perspective, soft shadows, and crisp label tabs on layer edges.

Avoid: fake 3D bevel charts, unreadable tiny labels, dramatic perspective that obscures the relation.

Prompt token: `isometric spatial hierarchy, 3-5 floating layers or shelves, clear vertical support, edge label tabs, soft shadows`.

### 23. Room / Stage / Workspace Diagram

Use for: work/life scenes, organizational roles, collaboration flow, daily routine, social dynamics.

Forms: cutaway room, top-down workspace, paper stage, control room, workshop bench, studio floor.

Best layout: one room or stage divided into functional zones. Each zone has a role, object, or state.

Beauty move: use a dollhouse cutaway or tabletop scene; labels attach to surfaces, doors, trays, screens, or floor zones.

Avoid: realistic office stock scene, decorative people, cluttered room full of props.

Prompt token: `clean cutaway room diagram with functional zones, object-first storytelling, attached surface labels`.

### 24. Terrain / Landscape Model

Use for: uncertainty, strategy terrain, emotional pressure, market fields, learning difficulty.

Forms: contour map, valley and ridge, weather front, pressure field, obstacle terrain, island chain.

Best layout: one visible route or viewpoint across terrain. The shape of the land carries the argument.

Beauty move: use soft contour lines, paper terrain layers, or miniature landscape; keep labels as map tags.

Avoid: fantasy landscape, over-detailed scenery, decorative atmosphere without information.

Prompt token: `miniature editorial terrain model with contour layers, one route or pressure field, map-tag labels`.

### 25. Orbit / Depth Network

Use for: attention, dependency distance, influence, ecosystem gravity, multi-agent systems.

Forms: orbital rings, depth lanes, nested spheres, gravity field, foreground-background dependency map.

Best layout: central anchor plus 2-4 distance rings or depth planes. Distance must mean something.

Beauty move: use thin orbit lines, soft depth fade, and clear cluster labels.

Avoid: decorative sci-fi solar system, random constellation, equal dots everywhere.

Prompt token: `depth network with central anchor, 2-4 semantic orbit rings, distance encodes relationship strength`.

### 26. Object Board / Physical Model

Use for: abstract concepts that need tactile understanding without characters.

Forms: tabletop model, block kit, magnetic board, folded paper model, index-card machine, modular toy-like system.

Best layout: objects arranged on a board or table with visible spatial relation: near/far, stacked, blocked, nested, connected.

Beauty move: make the model feel handmade and designed: soft edges, tiny tabs, physical connectors, calm material palette.

Avoid: random cute objects, toy-like without meaning, scattered props.

Prompt token: `tactile tabletop information model with blocks, folded cards, tabs, connectors, and spatial relationships`.

## Hybrid Rules

Good hybrids:

- flow plus mini scorecard: each stage has evidence or quality criteria
- map plus timeline: a journey unfolds over time
- pyramid plus cutaway: levels hide mechanisms
- matrix plus highlighted path: a decision has a recommended route
- storyboard plus diagram overlay: emotion and mechanism both matter
- spatial stack plus carrier path: hierarchy and workflow both matter
- room diagram plus small multiples: different roles occupy different zones
- service blueprint plus emotional curve: process and human state both matter
- causal model plus terrain: cause and pressure both matter
- Sankey plus scorecard: flow and quality criteria both matter

Bad hybrids:

- matrix plus ecosystem plus timeline in one image
- flow plus pyramid when neither dominates
- mini charts sprinkled around as decoration
- visible legend bigger than the focal form
- 3D perspective used only for spectacle
- Sankey without real quantity
- causal loop without evidence for causality

## Visual Figure / Guide Object

The skill may use a recurring visual figure when the user wants a stronger identity, but it should be a guide object, not a mascot character.

Good guide objects:

- Prism Lantern: a small faceted lamp that reveals structure with light paths
- Folded Compass: an origami-like pointer that marks the reader path
- Index Lens: a transparent rounded lens that frames the focal relation
- Signal Thread: a colored cord that moves through each diagram
- Anchor Tile: a small recurring tile used as the package mark

Rules:

- The guide object is optional and secondary.
- It must not have face, eyes, limbs, or mascot behavior.
- It should clarify the form: point to the path, illuminate a layer, hold a label, mark the focal node.
- It may recur across an article package as an identity mark.
- It must never replace the diagram form.

Prompt token: `optional recurring guide object: a small faceted Prism Lantern / Folded Compass / Index Lens, no face, no limbs, used only to reveal the diagram structure`.

## Beauty Rules For Diagram Forms

- Prefer rounded, hand-friendly, editorial shapes over hard square boxes by default.
- Use one large focal module, several smaller support modules, and a quiet zone.
- Use asymmetry unless the form is comparison, matrix, scorecard, or pyramid.
- Use whitespace as a reading tool: margins, gutters, pause zones, and quiet corners.
- Keep text containers integrated into the form: tabs, tags, labels, stamps, captions, shelf labels, map markers.
- Use 1-3 semantic colors. Same color must mean the same role.
- Avoid one-note palettes. Pair a calm base with one warm and one cool accent when possible.
- Replace generic arrows with carriers: ribbons, rails, cords, roads, pipes, shelves, contours, orbit lines, light trails.
- Do not make all elements equally cute. Charm comes from one small visual twist plus disciplined structure.
