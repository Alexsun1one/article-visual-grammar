# Connection Grammar

An article illustration works when the viewer understands how elements affect one another. Use semantic connectors, not decorative arrows.

## Connector Types

### Continuous Carrier

Shows the full reading path and creates hierarchy.

Use: ribbon, road, conveyor, river, thread, shelf, timeline, central spine.

Good for: almost any article body image with multiple ideas. Prefer this when the image must hold several information forms without becoming cluttered.

### Path

Shows movement through time or process.

Use: road, string, conveyor, river, route line, thread, dotted trail.

Good for: pipeline, map, journey, handoff.

### Transformation

Shows an object changing state.

Use: before/after material, stamp, oven, press, filter, lens, repair patch.

Good for: workflow, product process, learning, rewrite, distillation.

### Boundary

Shows permission, safety, inside/outside, acceptance/rejection.

Use: gate, membrane, door, threshold, customs desk, fence, lock, filter.

Good for: security, trust, policy, privacy, review.

### Containment

Shows organization, memory, ownership, grouping.

Use: drawer, jar, shelf, folder, capsule, museum case, box.

Good for: knowledge, archive, collections, taxonomy.

### Weight

Shows tradeoff, evidence, pressure, priority.

Use: scale, counterweight, heavy object, compressed spring, sinking platform.

Good for: decisions, incentives, risk.

### Repetition

Shows comparison or pattern.

Use: repeated panels, repeated objects, repeated icons with one variable changed.

Good for: options, versions, personas, modes.

### Causality

Shows cause/effect.

Use: domino, lever, fuse, root, gear, weather pressure, chain reaction.

Good for: analysis, explainers, failure modes.

## Label Grammar

Labels should attach to objects. Prefer:

- tag on object
- stamp mark
- sticky note
- tray label
- signpost
- small callout line
- caption under a panel

Avoid floating sentences. Use the smallest label count that preserves the idea. Simple diagrams often need 4-8 labels; maps, matrices, service blueprints, evidence tables, and field atlases may use 6-12 labels when the layout is built for scanning.

## Color Grammar

Use color semantically:

- red: warning, blocked, pain, loss, contradiction, result
- orange: movement, transition, active path, handoff
- blue: context, system signal, feedback, memory, trace
- green only when the article needs growth/success/health
- yellow only for highlight or attention, not as a background wash

Never use color merely to decorate.

## Three-Second Test

A viewer should be able to trace the idea in three seconds:

1. where the image starts
2. what changes
3. where the result or tension lands

If this fails, simplify the connector before changing style.

For multi-part images, run the carrier test: can the viewer see the main line first, then read the attached nodes? If not, add or strengthen the continuous carrier.
