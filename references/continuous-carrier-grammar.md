# Continuous Carrier Grammar

The strongest article illustrations often have a continuous visual carrier: one line, ribbon, path, conveyor, thread, river, shelf, road, timeline, or spine that moves through the whole image. This creates hierarchy.

Do not default to disconnected panels. Panels can exist, but they should hang from or be crossed by one continuous carrier.

## Hierarchy Model

Use four levels:

- **L0 Carrier**: the main line/ribbon/path. It tells the reader where to start and how to move.
- **L1 Anchor Nodes**: 3-5 main stops on the carrier. Each stop is a scene, object, or state.
- **L2 Embedded Information Forms**: small table, relationship map, mini chart, checklist, comparison card, stack, quote card, or label cluster attached to a node.
- **L3 Text Marks**: short labels, stamps, tags, callouts. These explain only what cannot be read visually.

If L0 is missing, the image often becomes a pile of metaphors.

## Carrier Types

### Ribbon Path

Best for: narrative, transformation, method, article argument.

Visual: a thick colored ribbon runs through objects. Nodes sit on the ribbon.

### Conveyor / Assembly Line

Best for: workflow, production, AI/process pipeline.

Visual: objects move on a belt and physically change at stations.

### Route Map

Best for: learning path, launch path, user journey.

Visual: path bends through landmarks. Use 4-6 stops max.

### Timeline Shelf

Best for: history, phases, before/after, product evolution.

Visual: horizontal shelf or ruler holds states in order.

### Spine With Branches

Best for: argument structure, article outline, layered reasoning.

Visual: central spine with 3-5 branches. Each branch contains an info form.

### River / Flow

Best for: information flow, attention, market movement, system dynamics.

Visual: stream carries objects; eddies or gates show filtering.

## Embedded Information Forms

Attach these to L1 nodes. Use 1-3 per image, not all.

- `mini table`: two-column comparison, checklist, pros/cons, before/after.
- `relationship map`: 3 nodes and 2-3 lines.
- `stack`: layers, priorities, dependency.
- `micro chart`: bar, line, dots, slope, progress.
- `quote card`: one short phrase or blank text zone.
- `decision card`: A/B fork with one cost mark each.
- `state badge`: pass/fail/risk/done.
- `object cutaway`: one object opened to show inner parts.

## Layout Rules

- The carrier must be visible within 1 second.
- Use 3-5 anchor nodes by default.
- The carrier should occupy 60%-85% of image width.
- Each node must have a distinct job: input, tension, transformation, check, result.
- Side information must attach to nodes with short connector ticks, not float randomly.
- Keep one large quiet area so the image does not become a poster of symbols.

## Text Rules

- Put L1 labels on or near anchor nodes.
- Put L2 labels inside embedded forms.
- Put L3 marks as stamps/tags/badges.
- Avoid a title unless the user asks.
- For publish mode, draw blank label zones along the carrier and overlay real text later.

## Prompt Formula

```text
continuous carrier:
carrier type:
start point:
anchor nodes:
embedded information forms:
side branches:
text zones:
end state:
```

## Anti-Patterns

- Four disconnected panels with no main line.
- Many symbols with no reader path.
- Equal-sized nodes with no primary/secondary hierarchy.
- Labels floating without attachment.
- Every node using the same information form.
- Cute character replacing the carrier.
