# Visual Structures

Pick structure first. Style comes later.

This file names the conceptual relation. After choosing one relation here, use `diagram-form-catalog.md` to choose the concrete visual form, then use `visual-token-system.md` to regulate colors, line weights, line patterns, labels, emphasis, and spatial depth.

Example: `stack` may become a pyramid, layered shelf, sediment stack, isometric layer tower, or room floors. `map` may become a route map, subway map, terrain model, or orbit-depth map.

## 1. Contrast / Before-After

Use for: transformation, critique, improvement, "why this matters".

Layout: left/right or top/bottom. Same frame, different state. Keep the changed variable obvious.

Prompt token: `same composition repeated twice, only the state changes`.

## 2. Pipeline / Assembly Path

Use for: workflows, content production, model pipeline, business process.

Layout: input -> operation -> output. Use 3-5 stages only.

Prompt token: `objects move through a visible path, each stage physically changes the object`.

Concrete forms to consider: linear flow, curved path, conveyor, handoff chain, branching decision flow, swimlane, tabletop conveyor.

## 3. Stack / Layers

Use for: systems, hierarchy, capability layers, causes under symptoms.

Layout: layered shelf, sediment, cake slice, building floors, transparent stack.

Prompt token: `layers are physically stacked, lower layers support upper layers`.

Concrete forms to consider: pyramid, stepped pyramid, layered stack, foundation tower, tier shelf, mountain contour, isometric spatial stack.

## 4. Boundary / Gate

Use for: permission, trust, safety, privacy, platform rules, approval.

Layout: outside -> checkpoint -> inside. Show one thing blocked or transformed.

Prompt token: `a visible boundary changes what can pass`.

Concrete forms to consider: gate transfer, membrane, customs desk, lock, filter, checkpoint, room threshold, terrain obstacle.

## 5. Map / Route

Use for: journey, learning path, migration, launch plan, narrative arc.

Layout: winding path with 3-6 landmarks. Avoid dense roadmaps.

Prompt token: `one path with landmarks, detours, and a visible destination`.

Concrete forms to consider: route map, island map, subway map, terrain contour, neighborhood map, migration path, miniature landscape.

## 5b. Continuous Carrier / Story Ribbon

Use for: article arguments that need hierarchy, multiple related ideas, a clear reader path.

Layout: one visible carrier line/ribbon/spine crosses the whole image. 3-5 anchor nodes attach to it; small tables, charts, relationship maps, or labels hang from the nodes.

Prompt token: `one continuous visual carrier with anchor nodes and embedded information forms`.

Concrete forms to consider: carrier ribbon, signal thread, rail, cord, orbit line, terrain route, tabletop connector.

## 6. Small Multiples

Use for: comparing options, patterns, failure modes, versions, personas.

Layout: 2-4 repeated mini-scenes with same scale and composition.

Prompt token: `repeat the same frame, change only one variable per panel`.

Concrete forms to consider: two-panel contrast, three-card comparison, 2x2 small multiples, same-frame scenario row, wrong-right pair.

## 7. Feedback Loop

Use for: iterative systems, learning loops, compounding, flywheels.

Layout: circular or looping path. Use one entry point and one output.

Prompt token: `loop has visible return path and accumulated residue`.

Concrete forms to consider: feedback loop, flywheel, spiral, loop with residue, circular service cycle, orbit ring.

## 8. Exploded View / Anatomy

Use for: explaining hidden components, product teardown, system internals.

Layout: main object disassembled into labeled parts.

Prompt token: `parts float apart just enough to reveal how it works`.

Concrete forms to consider: exploded view, cutaway block, cross-section, x-ray layer, parts tray, room cutaway.

## 9. Tradeoff Scale

Use for: decisions, cost/benefit, speed vs safety, quality vs quantity.

Layout: scale, seesaw, forked path, two trays, balance beam.

Prompt token: `two options visibly exert different weight or cost`.

Concrete forms to consider: 2x2 matrix, spectrum map, effort-impact grid, balance scale, ladder, pressure gauge.

## 10. Bottleneck / Failure Leak

Use for: bugs, constraints, blocked growth, hidden operational cost.

Layout: flow enters a narrow part, leaks, jams, or piles up.

Prompt token: `the failure is physical and local, not everywhere`.

Concrete forms to consider: funnel, sieve, filter stack, narrow bridge, pressure valve, queue bottleneck, leakage pipe.

## 11. Container / Archive

Use for: memory, knowledge management, saved context, collections.

Layout: shelves, drawers, jars, boxes, museum cases, filesystem cabinet.

Prompt token: `items are sorted into containers by meaning`.

Concrete forms to consider: evidence shelf, archive cabinet, nested containers, tabletop model, specimen board.

## 12. Role Theater

Use for: social dynamics, stakeholder conflict, user behavior, agent teams.

Layout: stage, table, meeting, market stall, chessboard, control room.

Prompt token: `each role has a clear position and prop`.

Concrete forms to consider: paper theater, storyboard, cutaway room, workspace diagram, market map, orbit system.

## Selection Rule

If one shot needs two structures, split it into two images. A strong article image usually has one structure, one diagram form, one visual token plan, one metaphor world, and one focal action.
