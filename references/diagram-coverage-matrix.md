# Diagram Coverage Matrix

Use this matrix as a backstop after `form-selection-engine.md`. It ensures the skill can cover work, life, systems, strategy, data, emotion, and spatial relationships without forcing everything into a flowchart.

This file is not a list of trendy chart names. It is a mapping from article meaning to visual form.

## Meaning Families

| Meaning family | Reader question | First forms | Simpler fallback | Accuracy trap |
| --- | --- | --- | --- | --- |
| Sequence | What happens first, next, last? | flow, timeline, journey map, storyboard | 3-step carrier | too many steps |
| Transformation | What changes the object/state? | cutaway, conveyor, filter, before/after | split object | decorative arrows |
| Hierarchy | What supports what? | pyramid, layered stack, tree, spatial stack | shelf layers | fake importance order |
| Choice | Which option is better? | matrix, scorecard, decision tree, scale | two-column contrast | unlabeled axes |
| Territory | Where are we moving? | route map, terrain, subway map, strategic landscape | stepping stones | random landmarks |
| System | What depends on what? | ecosystem, causal model, orbit, room zones | hub-spoke | hairball network |
| Evidence | Why believe this? | evidence shelf, mini table, small multiples, receipts | annotated proof object | fake precision |
| Quantity | How much moves or changes? | bar, dot, line, Sankey, waffle, histogram | direct labels | invented data |
| Overlap | What belongs to both? | Venn/Euler, shared zone, overlapping fields | 2-set contrast | meaningless overlap |
| Human situation | What does this feel like in use? | storyboard, room/stage, object theater | one object state | character drama unsupported by text |
| Work / life system | How do habits, roles, and tools interact? | room diagram, calendar map, service blueprint, object board | workspace split | cluttered office scene |
| Hidden mechanism | What is inside? | cutaway, exploded view, x-ray, anatomy | one peeled layer | too many tiny parts |
| Constraint | Where does it fail? | bottleneck, gate, membrane, pressure valve, terrain obstacle | pinched path | making risk look certain |
| Accumulation | What builds up? | loop with residue, stacked layers, timeline shelf, reservoir | before/after + residue | clean loop with no changed state |
| Identity / taxonomy | What kinds exist? | tree, nested shelf, field guide, specimen grid | 3-card comparison | category boundaries invented |

## Article Domains To Forms

### Work And Operations

Use for workflows, meetings, teams, handoffs, operations, processes, agents.

Best forms:

- service blueprint when visible and hidden work both matter
- swimlane when roles are central
- flow when sequence is central
- room/workspace diagram when responsibilities occupy zones
- bottleneck when failure is local
- evidence shelf when review or audit matters

Beauty move: make work physical through trays, rails, rooms, benches, calendars, tickets, or routes.

### Life And Habits

Use for routines, energy, attention, family, learning, emotional recovery.

Best forms:

- calendar map for repeated rhythm
- room/stage for zones of life
- terrain for difficulty and pressure
- spectrum for emotional intensity
- storyboard for human beats
- object board for sorting priorities

Beauty move: keep the scene quiet and spacious. A life diagram should feel humane, not like an enterprise dashboard.

### Product And Strategy

Use for product essays, market arguments, moat, positioning, pricing, GTM.

Best forms:

- matrix for positioning and tradeoff
- value chain for strategic dependency
- landscape map for competitive terrain
- scorecard for criteria
- causal model for growth/failure logic
- ecosystem for stakeholders

Beauty move: use semantic axes and one highlighted path or quadrant. Do not produce consultant box wallpaper.

### Technical / AI / Systems

Use for architecture, AI agents, security, automation, APIs, reliability.

Best forms:

- cutaway for hidden mechanism
- boundary/gate for permission, trust, safety
- service blueprint for frontstage/backstage agent work
- causal model for failure chain
- flow with typed carriers for data movement
- spatial stack for layered architecture

Beauty move: use technical clarity with soft editorial material. Make labels crisp and attached.

### Culture / Commentary

Use for social observation, values, media, belief shifts.

Best forms:

- two-state contrast
- spectrum or scale
- storyboard
- Venn/shared zone
- field map
- object metaphor

Beauty move: simple often wins. A precise object plus a state stamp can beat a complex system map.

## Data / Chart Coverage Backstop

Use real charts only when the article has data or a data-like comparison.

| Data relation | Prefer | Avoid |
| --- | --- | --- |
| trend over time | line, timeline, slope, milestone shelf | 3D chart |
| category comparison | bar, dot plot, lollipop, ranked slips | pie by habit |
| distribution | histogram, strip, box, violin, specimen row | average-only diagram |
| correlation | scatter, quadrant, field map | causal arrows |
| part-to-whole | stacked bar, waffle, nested container | too many pie slices |
| flow quantity | Sankey, alluvial, weighted ribbon | equal arrows with fake width |
| hierarchy / containment | tree, treemap, nested shelf, icicle | mind-map |
| geography / territory | map, route, contour field | decorative globe |
| schedule | timeline, calendar, Gantt-like strip | dense roadmap board |

If data is approximate or conceptual, use qualitative visual marks and avoid numeric precision.

## Forms To Add When Coverage Feels Thin

Consider these if the first plan feels repetitive:

- decision tree
- force-field diagram
- emotional curve
- swimlane
- calendar heat strip
- specimen / field-guide grid
- exploded view
- root-cause tree
- stock-and-flow reservoir
- queue / waiting-room diagram
- nested folder / archive
- affordance map
- risk register mini-table
- ladder with thresholds
- bridge / crossing
- control panel / cockpit

Use them only when they answer the reader's question better than a simpler form.

## Anti-Coverage Rule

Coverage is not maximalism.

Reject a candidate form if:

- it is chosen because the catalog has it, not because the article needs it
- it requires labels the article does not provide
- it implies data, order, hierarchy, or causality that the source does not support
- it makes the image harder to read in the first three seconds
