# Prompt Builder

Use this formula for one image at a time.

```text
Generate one standalone 16:9 horizontal article illustration.

Article context:
{one sentence about the article}

Figure spec:
{source anchor; package role; reader takeaway; misunderstanding prevented; simplest sufficient rung; output family; exact diagram form; complexity budget; typography mode; human presence level; charm level; visual harness style lineage; accuracy risk; must-not-show constraints}

Reader takeaway:
{what the reader should understand or feel after seeing the image}

Form selection:
{reader misunderstanding prevented; simplest sufficient rung; why this is not simpler; why this is not more complex; accuracy risk; complexity budget}

Clarity contract:
{one image / one claim; reader path; start-change-result anchors; object budget; label budget}

Visual structure:
{one structure from visual-structures.md}

Diagram form:
{family and specific form from diagram-form-catalog.md; why it fits; layout skeleton; spatial mode; what must be visible in three seconds; rejected alternatives; beauty move; optional guide object}

Coverage check:
{meaning family from diagram-coverage-matrix.md; why this form covers sequence / transformation / hierarchy / choice / territory / system / evidence / quantity / overlap / human situation / work-life system / hidden mechanism / constraint / accumulation / taxonomy}

Visual token system:
{color roles; line roles and widths; line patterns; shape roles; label containers; emphasis rule; spatial tokens if used}

Typography token system:
{draft label mode or publish overlay mode; Chinese/English font feeling; L1/L2/L3 type levels; label containers; text-free zones; real overlay font recommendation if applicable}

Human presence and charm system:
{H0-H5 human presence level; C0-C5 charm level; chosen form: none / human trace / role token / abstract faceless figure / storybook actor / explicit mascot; what the human element clarifies; forbidden character traits}

Action character system:
{none or one Paper Operator; operator family; article domain; action verb; object acted on; relation clarified; what breaks if removed; scale; forbidden drift}

Visual harness and style atlas:
{output family; form family; spatial mode; style lineage; abstraction level; material; line behavior; color system; composition engine; why this lineage helps comprehension; anti-copy boundary}

Image quality harness:
{quality target; material family; 6/8-column grid and generous margins; focal hierarchy; semantic palette; typography/overlay plan; texture/depth rule; negative space contract; anti-default-AI constraints}

Composition stability preset:
{one preset from composition-stability-presets.md; why it fits; fixed layout; max anchors; max labels; focal module; seven-word reader path; quiet zones; allowed depth; forbidden drift}

Metaphor world:
{one world from style-token-bank.md, chosen to fit the article}

Symbol system:
{PrismGrid / Anchor Glyph System or none; primitive geometry; symbol classes; semantic shape map; carrier glyph; label containers; color roles; layout skeleton; micro chart form}

Package symbol legend:
{for multi-image packages: shared primitive geometry; shared shape meanings; shared color meanings; shared label containers; recurring marks; inheritance rule}

Core objects:
{5-9 concrete objects, no vague concepts}

Connection grammar:
{path / transformation / boundary / containment / weight / repetition / causality}

Continuous carrier:
{carrier type from continuous-carrier-grammar.md; start point; 3-5 anchor nodes; embedded information forms; end state}

Line grammar:
{primary carrier line or ribbon; secondary connectors; callout style; line weight; direction; how labels attach}

Chart / diagram form:
{none / mini table / small multiples / micro chart / cutaway / route map / tradeoff scale / pyramid / matrix / timeline / loop / funnel / ecosystem / scorecard / part-to-whole / tree / Venn / Sankey / service blueprint / causal model / value chain / storyboard / canvas / spatial stack / room / terrain / orbit / object board; why it clarifies the article}

Hierarchy:
L0 carrier: {main line/ribbon/path/spine}
L1 anchor nodes: {3-5 main objects, states, or scenes}
L2 embedded forms: {small table / relationship map / micro chart / checklist / comparison card / cutaway}
L3 text marks: {labels, stamps, badges, short callouts}

Focal action:
{one physical action: sorting, squeezing, crossing, weighing, repairing, revealing, comparing, growing, blocking, routing}

Composition:
{where objects sit, where the viewer starts, how the eye moves, where the result lands, and how the chosen diagram form uses space, depth, and whitespace}

Labels:
{concise Chinese labels attached to objects; label count chosen by diagram form and cognitive load, usually 4-8 for simple diagrams and 6-12 for maps, matrices, service blueprints, field atlases, or technical plates}

Text plan:
{model-rendered labels or blank labels for overlay; L1 focal label; L2 object labels; L3 state marks; exact text zones; font feeling; text-free zones; render text now or leave blank}

Beauty gate:
{target score 4 or 5; whitespace plan; one focal surprise; what was removed to keep it simple; remaining risk}

Smoke target:
{what must read in three seconds; what would trigger regeneration; the single repair move to try first if it looks generic, PPT-like, or visually muddy}

Style tokens:
{base style + one style mode}

Comic/picture-book form:
{optional: clear-line picture-book explainer / 2x2 yonkoma explainer / picture-book spread / diagram comic}

Color semantics:
Ink means {primary outline/text}; Paper means {background}; Quiet means {secondary scaffolding}; Passage means {path/flow/route}; Evidence means {proof/data}; Risk means {constraint/failure/loss}; Growth means {accepted/output/recovery}; Human means {emotion/social pressure}; Focus means {the single chosen emphasis}. Use only needed roles.

Constraints:
Editorial, PrismGrid Symbol System, Paper Operators, instruction-manual, field-atlas, explanation-engineering, soft diagram infographic, or clear-line picture-book explainer style, not a hand-drawn mascot explainer. Treat the Figure spec as binding: do not add extra claims, extra data, extra labels, extra characters, or extra art styles beyond the complexity budget. Use the Visual Harness as binding: choose one style lineage, one material, one line behavior, one color system, and one composition engine. Use the Image quality harness as binding: name one material family, use a 6-column or 8-column invisible grid, keep generous margins, use one focal hierarchy move, reserve quiet zones, choose 3-5 semantic colors, and include an anti-default-AI list. Use the Composition stability preset as binding: choose exactly one preset, obey its fixed layout, stay within max anchors and max labels, make the focal module largest or clearest, preserve quiet zones, and forbid drift into a different layout. Use the Action character system only when a Paper Operator physically clarifies the relation; the operator must pull, frame, reveal, block, tune, fold, sort, mend, weigh, arrange, carry, shelter, distill, balance, light, archive, transform, stack, bridge, repair, or sweep a concrete object. If removing the operator would not weaken the diagram, do not draw it. Adapt the operator's tool to the article domain: art can use frames, palettes, plinths, light cards, crop marks, and swatches; culture/history can use archive folders, index cards, stage curtains, museum labels, and timelines; life/psychology can use folded notes, chairs, weather cards, envelopes, soft screens, and repaired paper edges; business/product can use routes, gates, scales, stacks, maps, and decision stamps; engineering can use carriers, nodes, cutaways, lenses, traces, loops, and failure bins. Do not force engineering gates, nodes, or workflow props into art, culture, personal, or emotional essays. Do not prompt "in the style of" a living artist or contemporary copyrighted illustrator. For historical art lineages, use transferable principles only: composition, line, color, material, geometry, and spatial rhythm. Do not copy a famous artwork composition or signature mark. Use the named diagram form as the composition engine: if flow, show a graceful carrier path; if pyramid, show soft supported layers; if matrix, show semantic axes; if map, show landmarks and territory; if loop, show return path and residue; if funnel, show input, constraint, output, and loss; if cutaway, show one opened object; if ecosystem, show clusters not a hairball; if scorecard, show criteria and evidence marks; if tree, show parent-child nesting; if Venn, show the meaningful intersection; if Sankey/alluvial, use weighted ribbons only for true quantity; if service blueprint, show route plus lanes; if causal model, show one main causal path and avoid arrow soup; if value chain, show strategic terrain and semantic axes; if storyboard, repeat framing and change one variable; if spatial stack, show isometric layers and edge labels; if room, show functional zones; if terrain, show contours and route; if orbit, show distance as meaning; if object board, show tactile spatial relations. Apply the visual token system: Ink for outlines/text, Paper for background, Quiet for secondary scaffolding, Passage for path/route/carrier, Evidence for proof/data marks, Risk for constraint/failure/loss, Growth for output/recovery, Human for emotional/social state only when needed, and one Focus accent only. Use Hairline quiet grid lines, Standard object outlines, one strong Carrier line, firm Boundary lines, and only one Emphasis stroke. Use solid lines for real relations, dashed for possible/future paths, dotted for evidence trails, broken for failures, wavy for pressure, and glow only for guide-object reveal. Use one visible continuous carrier line/ribbon/path as the hierarchy spine when the image has multiple parts, with 3-5 anchor nodes and 1-3 embedded information forms attached to it. Use aligned modules, clean object silhouettes, flat colors, structured label zones, chart-object hybrids where useful, and strong negative space. Same shape means same semantic role. Typography must follow the typography token system: render concise Chinese labels directly when integrated text improves the figure, using exact label text in clean editorial sans-serif feeling; use blank containers and later overlay only for dense, legally sensitive, client-critical, or editable text. Human presence must follow the H/C scale: H0-H1 for analytical/technical figures, H1-H2 for workflow and roles, H3 only when posture/action clarifies the idea, H4 only for storyboard or human consequence, H5 only by explicit user request. Do not default to hands, figures, or role tokens; add them only when they clarify agency, consequence, or decision. Prefer object-first metaphors, role tokens, or Paper Operators over generic stick figures. Paper Operators must be faceless folded-paper geometric figures with off-white body, charcoal outline, small semantic color tab, and one tool; no eyes, no mouth, no black blob body, no tiny thin legs, no stick-figure identity. Labels attach to objects, paths, containers, panels, surfaces, layer edges, floor zones, or state marks. If using a guide object, it must be a small Prism Lantern, Folded Compass, Index Lens, Signal Thread, or Anchor Tile with no face, eyes, limbs, or mascot behavior; it only reveals the diagram structure. If using PrismGrid, identity must come from semantic marks, five primitive geometries, carrier glyphs, label containers, color roles, and optional Paper Operators that operate the diagram, not from a mascot. Package symbol legend is internal: do not render a large visible legend box unless explicitly requested. Avoid courseware/process-slide habits: no checkmark under every step, no bottom principle bar, no long explanatory footer, no sterile row of equal cards. Use one focal node, one local transformation/failure mark, and one embedded artifact or micro chart. No top-left title unless explicitly requested. No PPT infographic look. No generic AI output: no stock icons, no floating circuit head, no decorative gradient blobs, no generic dashboard card wall, no fake tiny text texture, no robot/brain/rocket/lightbulb unless specifically requested. No Xiaohei-like black figures, no white-dot-eye mascot, no recurring tiny worker, no character standing beside the diagram, no head-and-shoulders user pictograms, no user avatars, no silhouettes, no generic stick-figure identity, no casual whiteboard doodle identity, no dense anime/manga style. Do not explain every detail. Make the idea clear in three seconds, but keep the metaphor visually memorable.
```

## Publish-Ready Chinese Text Prompt Add-On

Use this when exact Chinese typography matters:

```text
Use this only when overlay is deliberately chosen. Do not render final Chinese text. Draw clean blank label areas attached to the relevant objects: one focal label area, 2-7 object label areas, and 1-4 small status mark areas. Leave enough white space inside each area for later Chinese typography overlay. Keep the label areas integrated into the scene as tags, signs, stamps, tray labels, or panel captions.
```

## Figure Spec Prompt Add-On

Use this before the image prompt when the source is a full article:

```text
Before generating, use this compact figure spec as the source of truth:
- source anchor: {section / paragraph / quote}
- package role: {thesis anchor / mechanism / tension / evidence / human consequence / summary}
- reader takeaway: {one sentence}
- misunderstanding prevented: {one concrete misunderstanding}
- simplest sufficient rung: {single object / two-state contrast / simple diagram / spatial model / package system}
- output family: {article body figure / diagram and infographic / editorial and social / technical and product}
- form: {family / specific form / spatial mode}
- complexity budget: {1 claim, 3-5 anchor nodes or zones, 5-9 objects, label budget by diagram form, 1 dominant accent, 1 form}
- typography: {draft labels or publish overlay; font feeling; L1/L2/L3 zones}
- human presence: {H0-H5; C0-C5; none / trace / role token / faceless figure / storybook actor; what it clarifies}
- action character: {none / Paper Operator family; article domain; action verb; object acted on; what breaks if removed}
- visual harness: {output family; form family; spatial mode; style lineage; abstraction; material; line behavior; color system; composition engine; anti-copy boundary}
- image quality harness: {quality target; material family; grid/margins; focal hierarchy; palette; typography/overlay plan; negative space; anti-default-AI constraints}
- composition stability preset: {preset; fixed layout; max anchors; max labels; focal module; reader path; quiet zones; forbidden drift}
- accuracy risk: {what must not be overclaimed}
- must not show: {cliches, forbidden symbols, wrong forms}
```

## Image Quality Harness Add-On

Use this inside every final generation prompt unless the user is only asking for planning:

```text
Image quality harness:
- quality target: publish-grade article body figure, beauty score 4+.
- material family: {editorial paper model / field atlas plate / soft diagram infographic / spatial editorial model / clear-line picturebook explainer}.
- composition: 6/8-column invisible grid, generous margins, one large focal module, clear gutters, quiet zones.
- focal hierarchy: {largest object / strongest carrier / isolated module / one accent / cutaway reveal / guide-object light}.
- palette: semantic roles only; use 3-5 visible colors.
- typography: {model-rendered concise labels / blank overlay containers}; labels attached to objects, paths, surfaces, zones, or tabs.
- texture and depth: one controlled material; depth only if it encodes hierarchy, containment, route, or distance.
- negative space: leave {specific zones} quiet; remove {specific complexity}.
- anti-default-AI: no PPT, no stock icons, no robot/brain/rocket/lightbulb, no mascot identity, no fake dense text, no decorative arrows.
```

## Composition Stability Add-On

Use this inside every final generation prompt after the image quality harness:

```text
Composition stability preset:
- use exactly one preset: {Hero Object With State Marks / Carrier Through Gate / Two-State Contrast / Shelf Matrix / Field Atlas Plate / Cutaway Mechanism / Route Map With One Obstacle / Spatial Stack / Three-Beat Storyboard}.
- obey the fixed layout; do not invent a new layout.
- max anchors: {number}; max labels: {number}.
- focal module: {one thing}; make it largest or clearest.
- reader path: {seven words or fewer}.
- quiet zones: {where empty space must remain}.
- forbidden drift: {what the model must not add}.
```

## Action Character Add-On

Use this only when a character is necessary to clarify the relation:

```text
Action character system:
- use one Paper Operator, not a mascot.
- operator: {Thread Runner / Lens Keeper / Gate Builder / Frame Setter / Color Tuner / Light Catcher / Archive Tender / Care Folder / Boundary Keeper / Weather Reader / Stack Mason / Residue Sweeper / Tradeoff Weigher}.
- article domain: {art / culture / life / psychology / business / product / engineering / education / finance / other}.
- action: {pull / frame / reveal / block / tune / fold / sort / mend / weigh / arrange / carry / shelter / distill / balance / light / archive / transform / stack / bridge / repair / sweep}.
- the operator physically operates {carrier / lens / gate / frame / palette / light card / archive folder / note / boundary screen / weather card / layer / map / evidence card}; it cannot stand beside the diagram.
- appearance: faceless folded-paper geometric figure, off-white body, charcoal outline, small semantic color tab, no eyes, no mouth, no black blob body, no tiny legs, no stick-figure identity.
- scale: secondary to the diagram, usually 8-18% canvas height.
- what breaks if removed: {the relation becomes less clear}.
```

## Prompt Repair Moves

If the image is boring:

```text
Regenerate with a stronger physical metaphor. Replace generic icons and arrows with concrete objects from the chosen metaphor world. Keep one focal action and one visible connection grammar.
```

If the image is confusing:

```text
Regenerate with fewer objects. Use one clear start point, one transformation or boundary, and one result. Keep labels attached to objects.
```

If the image has no hierarchy:

```text
Regenerate with one continuous carrier line/ribbon/path running through the whole image. Attach 3-5 anchor nodes to the carrier. Put small tables, relationship maps, charts, or labels as side branches from nodes. Make the carrier visible first.
```

If the visual tokens are inconsistent:

```text
Regenerate using the Visual Token System. Assign semantic color roles before drawing: Ink, Paper, Quiet, Passage, Evidence, Risk, Growth, Human, Focus. Make Passage the carrier color, Risk only the constraint/failure color, Evidence only proof/data marks, and Focus a single accent. Use Hairline/Standard/Carrier/Boundary/Emphasis line hierarchy. Put all labels in attached containers.
```

If the image looks like PPT:

```text
Regenerate as an editorial grid-collage image, not a PPT diagram and not a hand-drawn mascot scene. Replace generic boxes and arrows with structured modules, paper-cut objects, chart-object hybrids, paths, containers, gates, or repeated mini-scenes.
```

If the image uses the wrong diagram form:

```text
Regenerate with the correct form from the diagram catalog. State the family and specific form first, then rebuild the composition around it. Replace the current layout with {flow / pyramid / matrix / map / timeline / loop / funnel / cutaway / ecosystem / small multiples / scorecard / part-to-whole / spectrum / storyboard / canvas / spatial stack / room / terrain / orbit / object board}. Keep the article takeaway, but change the spatial logic.
```

If the image is ugly:

```text
Regenerate with stronger grid alignment, more negative space, one large focal module, fewer labels, and a restrained editorial color palette. Use rounded, hand-friendly, editorial shapes rather than hard square boxes unless the article specifically needs technical rigidity.
```

If the image is visually busy but unclear:

```text
Regenerate using the clarity budget: one reader takeaway, one visual structure, one metaphor world, one focal action, one connector grammar, 5-9 objects, and only the labels needed by the diagram form. Make start, change, and result visible.
```

If the image is overcomplicated:

```text
Regenerate using the simplest sufficient rung from the Form Selection Engine. Try single object metaphor, two-state contrast, or 3-step carrier before using a spatial model. Remove any element that does not prevent a reader misunderstanding.
```

If the image has no cleverness:

```text
Regenerate with one truthful focal surprise from the Aesthetic Quality Bar: reveal a hidden layer, make the carrier become a bridge/thread/river, show residue after the loop, let the risk wedge physically pinch the flow, or use a Prism Lantern to illuminate only the decisive node. Do not add decorative charm.
```

If the image needs more warmth:

```text
Regenerate with the Human Presence and Charm System. Choose the minimum useful level: H1 human trace, H2 role token, H3 faceless abstract figure, or H4 storybook actor only if the article is about a human situation. Add C1-C2 warmth through rounded modules, tactile labels, and one truthful detail. Do not add a mascot or generic stick figure.
```

If the image needs a stronger style harness:

```text
Regenerate using the Visual Harness and Style Atlas. Choose one compatible lineage such as Swiss typographic clarity, Bauhaus constructive workshop, De Stijl grid, Matisse cutout logic, ukiyo-e-informed composition, Klee symbolic micro-world, Albers color interaction, Calder mobile balance, or risograph editorial print. Extract only composition, line, color, material, and spatial principles. Do not imitate a specific artist or artwork.
```
